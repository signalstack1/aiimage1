Now add the following properly with Supabase persistence.

1. Categories

Allow the admin user to create their own product categories.

Each category should be saved in Supabase.

The admin should be able to:

* Create category
* Edit category
* Delete category
* Assign category to a product

Also allow optional sub-categories.

Example:

Category: Clothing
Sub-category: Tops
Sub-category: Hoodies
Sub-category: Joggers

The product upload/edit page should let the admin select a category and, if available, a sub-category.

2. Product dropdown options / variants

Allow the admin to optionally create a dropdown selection for a product.

This should be fully custom and controlled by the admin.

Example:

Dropdown name: Size
Options: Small, Medium, Large, XL

Another example:

Dropdown name: Colour
Options: Black, White, Grey

The admin should be able to:

* Add dropdown name
* Add multiple dropdown options
* Save options to Supabase
* Attach the dropdown to a product
* Edit or remove dropdowns
* Leave dropdown blank if the product does not need one

On the public product page, if a product has a dropdown attached, show it clearly so the buyer can select an option before clicking the buy button.

3. Tags

Allow the admin to create their own product tags.

Tags should be saved in Supabase.

The admin should be able to:

* Create tag
* Edit tag
* Delete tag
* Assign multiple tags to a product

New
Best Seller
Limited Offer
Digital Product
Physical Product
Premium

Tags should be reusable across products.

4. Quantity / stock control

Add stock control to products.

Each product should have a stock quantity field.

The admin should be able to set the available quantity.

Also add a toggle:

Show stock quantity to customers: ON / OFF

If ON, the public product page should show something like:

“Only 5 left in stock”

If OFF, the product page should not show the exact stock number.

Also add stock status logic:

* If stock is greater than 0, product is available
* If stock is 0, product should show “Out of stock”
* If out of stock, disable the buy button or clearly prevent purchase
