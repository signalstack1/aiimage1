Please update the TVC website so it has two clearly defined monthly membership plans:

TVC Basic
TVC Plus

The website must also offer optional TVC van sticker packs as separate one-time add-ons available to members of either plan.

Before making changes, inspect the current pricing page, application flow, account creation flow, payment-link setup, member dashboard, public profiles, admin controls and Supabase database structure.

10. TVC PLUS IMAGE-UPLOAD LIMIT

TVC Plus members may upload up to 20 new work images per calendar month.

The system must:

Count successful uploads during the current calendar month
Display how many uploads have been used
Display how many uploads remain
Prevent upload number 21 during the same calendar month
Reset the allowance at the start of the next calendar month
Keep previously uploaded images visible
Keep image descriptions attached to the correct image

Deleting an image must not restore the monthly upload allowance.

This prevents users from repeatedly uploading and deleting images to bypass the limit.

Failed or abandoned uploads should not count against the allowance.

The upload limit must be enforced securely in the backend, not only in the user interface.

27. ARCHITECTURE RULES

These rules are mandatory:

Replit is the builder and development environment only.
Supabase is the production database, authentication provider and persistent backend source of truth.
Supabase Storage holds production uploads.
Vercel hosts the production frontend.
GitHub remains the source repository where applicable.
No production feature may require Replit to remain online.
No production API request may point to localhost, replit.dev, repl.co or another Replit runtime.
No production data may be stored permanently in the Replit filesystem.
No Supabase service-role key may be exposed in frontend code.
No secret key may be placed in a VITE_ or NEXT_PUBLIC_ variable.
Public Supabase browser access must use the appropriate anon or publishable key together with secure RLS policies.
Secure privileged operations must run server-side.

Update the repository architecture documentation so future Replit work continues to follow these rules.
